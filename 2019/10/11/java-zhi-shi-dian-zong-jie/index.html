第二章面向对象

1.面向对象：面向对象思想就是把一个项目，一件事情分成更小的项目，每一部分负责什么功能，最后再有这些部分组合成一个整体。

2.类的创建

    语法：public class 类名{}

​      File--->new--->class--->类名

​

3.创建对象

类名 对象名=new 类名();

​      对象名.方法名(); 对象名.属性

4.构造方法：方法名和类名相同,没有返回值

     public class 类名{

​         public 类名(){}//主要的做用是初始化操作

​      }

5.构造方法重载

（1）构造方法的重载：方法名与类名相同，参数列表不同

（2）方法的重载：在同一个类中包含了两个或者两个以上的方法，它们的方法名相同，方法的参数个数，顺序，类型不同，则称这个方法被重载了，这个过程称为方法的重载。

（3）构造方法的作用：用来实现多种初始化行为。

    public class 类名{

​         public 类名(){}

​         public 类名(String name,int age){}

​      }

6.构造方法的执行

     new 类名();--->执行构造方法

​      super();--->执行了父类的构造方法

7.static静态修饰符

（1）含义：static可以用来修饰属性，方法，和代码块。static修饰的变量属于这个类所有，即由这个类创建的所有对象共用一个static变量。通常把static修饰的属性和方法称为类属性（也可以称为类变量），类方法。不使用static修饰的属性和方法属于单个对象，通常称为实例属性，实例方法。

（2）要点：

​ 1.在加载类的过程中，完成静态变量的内存分配，再执行静态代码块，两者是在创建对象之前执行的。

​ 2.类属性和类方法可以通过类名和对象名访问，实例属性和实例方法只能通过对象名访问。

​ 3.类方法只能访问类属性和其他类方法，

注意：静态方法中不能使用this和super关键字。

8.java内存（栈和堆，static，方法区，gc（垃圾回收机制））

9.八种基本数据类型：

    byte  short   int   long

    float   double   Boolean   char

第三章面向对象三大特征—封装

第三章 面向对象三大特征—封装

1.封装的概念：封装是指将对象的状态信息隐藏在对象内部，不允许外部程序直接访问对象内部的信息，而是通过该类所提供的方法来实现对内部信息的操作访问。

2.封装的步骤

​ 1.把属性私有化（private）

​ 2.对这些属性设置get和set方法

​ 3.调用处理这些方法

3.封装的作用与好处：

作用：避免一些用一些不合乎常理的操作

好处：

（1） 提高了安全性

（2） 提高了复用性

（3） 隐藏了实现细节

4.封装的案例:

/* 文件名: EncapTest.java */

public class EncapTest{



   private String name;

   private String idNum;

   private int age;



   public int getAge(){

​      return age;

   }



   public String getName(){

​      return name;

   }



   public String getIdNum(){

​      return idNum;

   }



   public void setAge( int newAge){

​      age = newAge;

   }



   public void setName(String newName){

​      name = newName;

   }



   public void setIdNum( String newId){

​      idNum = newId;

   }

}

/* F文件名 : RunEncap.java */

public class RunEncap{

   public static void main(String args[]){

​      EncapTest encap = new EncapTest();

​      encap.setName("James");

​      encap.setAge(20);

​      encap.setIdNum("12343ms");



​      System.out.print("Name : " + encap.getName()+ 

​                             " Age : "+ encap.getAge());

​    }

}

5.修饰符

​ public—>公共的：在一个项目中都可以访问，权限最高

​ private—>私有的：只能在本类中进行访问，权限最低

​ protected—>安全的，受保护的：同一个包能进行访问，继承中之类可以访问父类

​ default—>默认的：同一个包能访问

第四章 面向对象三大特征—继承

1.继承的概念：父亲儿子

2.继承的关键字：extends

3.继承的好处：

（1）提高了代码的重用性

（2）提高了代码的可扩展性

3.继承的语法：

     public class 类名  extends 父类{    代码块   }

4.继承的使用与说明：

（1）使用：

可以直接调用父类的非私有属性和方法

​ 可以用super.方法 super.属性

（2）说明： Java中的继承只能单继承，

5.父类构造方法调用：super();

​ //super指代的是父类对象，this指代的是本类对象

​ 调用父类的构造方法必须在之类中调用

​ 必须实在之类构造方法的第一行代码

6.继承中的构造方法要点：

（1）执行子类的构造方法，会默认执行父类的无参构造方法

（2）当子类没有无参构造方法，但是有有参构造方法,子类的构造方法需手动添加

（3）父类没有定义无参构造方法，定义了带参构造方法，子类的构造方法必须调用父类的带参构造方法。

（4）无论是super还是this调用构造方法，只能放在第一行，构造方法只能调用一个

（5）构造方法只能在构造方法中调用

7.重写@Override

​ 必须继承

​ 方法名一致，参数个数，顺序，类型全部一致

8.重载与重写的区别：

重载:一定发生在同一类中，方法名相同，参数不同（类型不同，数量不同）

重写：一定发生在父类和子类中，方法名相同，参数相同，方法体不同

第五章 面向对象—多态

1.多态的概念：不同对象对同一消息产生不同的响应。

2.多态的必要条件

（1）继承

（2）重写

（3）父类引用指向子类对象（向上转型）

3.多态的优点：

（1）消除类型之间的耦合关系

（2）可替换性

（3）可扩展性

（4）接口性

（5）灵活性

（6）简化性

3.多态的使用

（1）第一个例子

 public class Father{

​         public void cooking(){

​            System.out.println("米饭");

​         }

​      }

​      public class Son extends Father{

​         @Override

​         public void cooking(){

​            System.out.println("熬粥");

​         }

​      }

​      public class Test {

​         public static void main(String[] args) {

​            Father son=new Son();

​            son.cooking();

​         }

​      }

（2）第二个例子

 public class Test {

​    public static void main(String[] args) {

​      show(new Cat());  // 以 Cat 对象调用 show 方法

​      show(new Dog());  // 以 Dog 对象调用 show 方法

​                

​      Animal a = new Cat();  // 向上转型  

​      a.eat();               // 调用的是 Cat 的 eat

​      Cat c = (Cat)a;        // 向下转型  

​      c.work();        // 调用的是 Cat 的 work

  }  

​            

​    public static void show(Animal a)  {

​      a.eat();  

​        // 类型判断

​        if (a instanceof Cat)  {  // 猫做的事情 

​            Cat c = (Cat)a;  

​            c.work();  

​        } else if (a instanceof Dog) { // 狗做的事情 

​            Dog c = (Dog)a;  

​            c.work();  

​        }  

​    }  

}

​    abstract void eat();  

}  



class Cat extends Animal {  

​    public void eat() {  

​        System.out.println("吃鱼");  

​    }  

​    public void work() {  

​        System.out.println("抓老鼠");  

​    }  

}  



class Dog extends Animal {  

​    public void eat() {  

​        System.out.println("吃骨头");  

​    }  

​    public void work() {  

​        System.out.println("看家");  

​    }  

}

4.向上转型（子类向父类的转换） 

 语法 ：

​      <父类型>  <引用变量名> = new  <子类型>（）；

​      例如：Animal an1=new Dog（）；

5.向下转型（必须基于向上转型）

    例如：Son son=(Son)father;

（1）属性无多态这个属性，只看“=”左边的类型

（2）编译时，能“.”出来，能调用的看“=”左边

8.异常的处理（Try-catch-finally块）：

语法：

第六章 抽象

1.抽象的概念:当一个类没有足够的信息去描述一个具体的对象，那么我们称之为抽象类。

2.抽象的关键字

​ abstract

​ 可以修饰类，可以修饰方法

3.抽象类的作用：

（1）抽象类是用来继承的

（2）抽象类无法new出对象

4.抽象的用法

     //抽象类

​      public abstract class 类名{

​         //抽象方法没有方法体

​         public abstract void 方法名();   

​      }
     //当父类是抽象类的时候，子类必须重写父类里面的抽象方法

​

5.final修饰符：最终的

修饰变量，修饰方法，修饰类

（1）.当修饰变量时，变量转换成常量，常量的值不能改变

​ final int a=18;//a=18;不能改变,必须给一个初始值

（2）.当修饰方法时，该方法不能被重写

（3）.当修饰类时，该类不能被继承

第七章 接口

为什么要用接口？：因为JAVA只支持单继承。

  1. 接口的概念

2.接口的关键字

​ interface

3.接口的语法

      public interface 接口名{}

​      File--->new--->interface//创建接口

4.接口的用法

      实现接口的关键字  implements

​      public interface 接口名{

​         //接口是纯抽象类

​         //接口里面的方法都是抽象方法

​         void setData();

​      }

​      public class Play implements 接口名{

​         @Override

​         void setData(){

​         }

​      }

​      public class Test {

​         public static void main(String[] args) {

​            Play play=new Play();

​            play.setData();

​         }

​      }

​      注意接口可以进行多继承

​      public interface 接口名 extends 接口名，接口名{}

​      一般用法

​      public class 类名 extends 父类 implements 接口，接口{}

多态与接口的区别：

（1） 多态：难

（2） 接口：简单，不能随意定义属性和方法，只能定义抽象方法，并且实现对口必须重写。

第八章 集合框架

1.ArrayList

1.用法

     ArrayList list=new ArrayList();

    list.add();//添加数据

    list.get();//获取数据

2.Map的用法：

       Map map=new HasMap();

​      map.put(key,value);//key不能重复

​      map.get(key);//获取key对象的value

3.泛型

      ArrayList<String> list=new ArrayList><String>();

      Map<String,String> map=new HasMap<String,String>();

      ArrayList<Map<String,String>> list=new ArrayList><Map<String,String>>();
